# researcher-deskless

> A portable Docker environment for researchers. Carry your full note-taking and reference management setup on a USB drive and run it on any Ubuntu machine.

## What's inside

| Tool | Purpose |
|------|---------|
| [Zotero](https://www.zotero.org/) | Reference manager — collect, organize, cite |
| *(coming soon)* [Obsidian](https://obsidian.md/) | Knowledge base — notes, links, ideas |

All data lives on the drive. All tools run in Docker. No installation required on the host beyond Docker itself.

---

## Requirements

Every host machine needs:

- Ubuntu 22.04 or later
- Docker installed
- An X11 display (standard on Ubuntu with Xorg)

```bash
sudo apt install docker.io
sudo usermod -aG docker $USER
```

> Log out and back in after adding yourself to the `docker` group.


## Getting started

### 1. Clone the repository onto your USB drive

```bash
git clone https://github.com/yourusername/researcher-deskless.git /media/$USER/your-drive/
cd /media/$USER/your-drive/
```

### 2. Build the Docker image

Do this once on any machine with internet access:

```bash
make build
```

This builds the image and saves it as `zotero.tar.gz` on the drive.

### 3. Run

On any Ubuntu machine:

```bash
make run
```

If the Docker image is not yet loaded on that machine, it will be loaded automatically from `zotero.tar.gz`. Zotero will open with your full library.

## How it works

The container starts as root, creates a user matching your host UID/GID, fixes ownership of the data directory, then drops privileges before launching Zotero. This ensures your library files on the drive are always owned by the correct user regardless of which machine you plug into.

WebRender and GPU acceleration are disabled so Zotero runs reliably without requiring a GPU or specific graphics drivers on the host.

```
USB drive
├── zotero.tar.gz  ──────────────►  docker load
└── zotero-data/   ──── -v mount ►  /zotero-data  (inside container)
                                          │
                                     Zotero 7
                                          │
                                     X11 display  ──►  your screen
```

## Data and privacy

All your data stays on the drive:

- Zotero library, attachments, and PDF annotations → `zotero-data/`
- Obsidian vault → `obsidian-data/` *(coming soon)*

Nothing is written to the host machine except Docker's image cache (which is safe to clear).
